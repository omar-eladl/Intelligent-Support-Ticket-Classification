# Vodafone Egypt RAG Chatbot

## Overview

This project is a Retrieval-Augmented Generation (RAG) chatbot designed for Vodafone Egypt customer support. Instead of relying solely on a Large Language Model (LLM), the system retrieves the most relevant information from a Vodafone knowledge base and uses it to generate accurate, context-aware responses.

The chatbot is designed to minimize hallucinations by ensuring that every answer is generated only from the retrieved documents.

---

## Features

* Hybrid Retrieval System

  * TF-IDF Retriever
  * Semantic Retrieval using multilingual-e5-base
* Retrieval-Augmented Generation (RAG)
* Arabic customer support
* Context-aware question answering
* Hallucination reduction through grounded responses
* Streamlit web interface
* Evaluation using Information Retrieval metrics

* Internet packages
* Call packages

* Recharge services
* Vodafone Cash
* Roaming
* Offers

The knowledge base is indexed for efficient retrieval before passing the relevant context to the language model.

---

## Retrieval Pipeline

The system combines two retrieval methods:

### 1. Sparse Retrieval

TF-IDF is used to retrieve documents based on keyword similarity.

### 2. Dense Retrieval

Sentence embeddings generated by multilingual-e5-base are used for semantic search, allowing the chatbot to understand the meaning of user queries rather than relying only on exact keyword matches.

### 3. Hybrid Retrieval

The scores from both retrievers are combined to produce more accurate document ranking.

---

## Generation Pipeline

After retrieving the most relevant documents:

1. The retrieved documents are merged into a single context.
2. A structured prompt is created.
3. The prompt instructs the LLM to answer only using the provided context.
4. Qwen2.5-3B-Instruct generates the final response.

This approach significantly reduces hallucinations and improves factual accuracy.

---

## Evaluation

The retrieval system was evaluated using standard Information Retrieval metrics:

* Recall@1
* Recall@3
* Recall@5
* Mean Reciprocal Rank (MRR)

These metrics were used to compare different retrieval approaches and select the best-performing configuration.
